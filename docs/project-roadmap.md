# AI 建筑可视化项目思路

## 项目目标

构建一个端到端的建筑可视化 Pipeline：**草图 → AI 渲染图 → 3D 模型 → Web 交互展示**

## 核心流程

```
┌─────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 建筑草图 │ → │  AI 建筑效果图 │ → │ 3D Gaussian │ → │  Web 可视化  │
│ (Sketch)│    │  (Rendering)│    │  Splatting  │    │  + 材质信息  │
└─────────┘    └─────────────┘    └─────────────┘    └─────────────┘
    │               │                  │                  │
    ▼               ▼                  ▼                  ▼
 手绘/扫描    Midjourney/       Luma AI/           Three.js + React
 平板绘制     通义万相          本地 GS 训练         交互界面
```

## 阶段拆解

### 阶段一：草图 → AI 建筑效果图

**输入**：用户手绘的建筑草图（照片/扫描件）
**输出**：高质量建筑渲染图

**关键决策**：
| 方案 | 优点 | 缺点 | 推荐场景 |
|-----|------|------|---------|
| **Midjourney** | 效果精美，建筑理解强 | 需科学上网，无官方 API | 追求品质，手动操作 |
| **通义万相** | 国内可用，中文支持好 | 效果略逊于 MJ | 国内项目，快速验证 |
| **Stable Diffusion** | 可控性强，本地部署 | 需 GPU，调参复杂 | 批量处理，深度定制 |

**技术要点**：
- 使用 ControlNet 保持草图结构
- Prompt 模板化（风格 + 材质 + 环境 + 光照）
- 输出多视角备选图

---

### 阶段二：效果图 → 3D Gaussian Splatting

**输入**：AI 生成的建筑效果图（单张或多张）
**输出**：.splat 或 .ply 格式的 3D 模型

**关键决策**：
| 方案 | 成本 | 精度 | 速度 | 推荐 |
|-----|------|------|------|------|
| **Luma AI** | $5/次 | 高 | 10min | MVP 验证首选 |
| **Wonder3D + GS** | 免费 | 中 | 30min | 单图重建，本地运行 |
| **DUSt3R + GS** | 免费 | 中高 | 20min | 多图重建 |
| **本地 GS 训练** | GPU 成本 | 最高 | 1-4h | 最终部署方案 |

**技术要点**：
- 单图重建用 Wonder3D 生成多视角
- 多图重建用 COLMAP 估计位姿
- 输出压缩为 .splat 格式（体积减小 10x）

---

### 阶段三：3D 模型 → Web 可视化

**输入**：Gaussian Splatting 模型文件
**输出**：交互式 Web 展示界面

**核心功能**：
1. **3D 展示**
   - 加载 .splat/.ply 文件
   - 旋转/缩放/平移交互
   - 材质高亮/切换

2. **材质信息面板**
   - 识别建筑材质（混凝土/玻璃/钢材等）
   - 展示材质属性（环保性/耐久性/成本）
   - 点击模型部位显示对应材质

3. **环境信息**
   - 地图定位
   - 气候数据（温度/湿度/日照）
   - 周边环境描述

**技术栈**：
- 框架：React + TypeScript
- 3D 渲染：Three.js + @react-three/fiber
- GS 加载：@mkkellogg/gaussian-splats-3d
- UI 组件：Ant Design / Chakra UI
- 地图：react-leaflet

## 数据流

```
用户上传草图
    │
    ▼
[POST] /api/v1/generate
    │
    ├──▶ 调用 AI 生图 API
    │
    ◀── 返回建筑效果图 URL
    │
用户确认/选择效果图
    │
    ▼
[POST] /api/v1/reconstruct
    │
    ├──▶ 下载效果图
    ├──▶ 生成多视角（单图→Wonder3D）
    ├──▶ 训练 Gaussian Splatting
    │
    ◀── 返回 .splat 文件 URL
    │
    ▼
前端加载 3D 模型
    │
    ├──▶ 渲染 Gaussian Splatting
    ├──▶ 显示材质面板
    └──▶ 展示地点信息
```

## 关键问题与对策

| 问题 | 对策 |
|-----|------|
| 单图 3D 重建精度低 | 先用 Luma AI 验证，后期换多图+COLMAP |
| GS 文件体积大 | 使用 .splat 压缩格式，前端分片加载 |
| 材质识别不准确 | 结合 GPT-4V + 建筑材质知识库 |
| Web 渲染卡顿 | WebGL2 + 视锥剔除 + LOD |
| API 成本过高 | 逐步替换为开源本地方案 |
